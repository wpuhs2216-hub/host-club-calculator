# v1.3.0 仕様書 — 新規伝票ダイアログ化・LOページ2列レスポンシブ

## 概要

新規伝票作成をモーダルダイアログ1画面に統合し、スワイプ不要の高速入力を実現。  
LOページはタブレット等の大画面で2列表示に対応。

---

## 1. 新規伝票ダイアログ（SlipWizard → NewSlipDialog）

### 1-1. 概要
5ステップのウィザードを廃止し、**1つのモーダルダイアログに全必須項目をまとめる**。  
客種・入店時間・同伴を1画面で設定し、[作成] で即座にタブ切替モードへ。

### 1-2. ダイアログレイアウト

```
┌──────────────────────────────┐
│  新しい伝票            [✕]  │
│                              │
│  ── 客種 ──                  │
│  [新規]   [R]   [正規]      │
│  [チケ有] [チケ無]           │  ← R選択時のみ
│  [¥0][¥1k][¥3k][¥5k]       │  ← 新規選択時のみ
│                              │
│  ── 入店時間 ──              │
│  [20][21][22][23][24]        │  ← 時
│  [0_][1_][2_][3_][4_][5_]   │  ← 分十の位
│  [0][1][2][3][4]             │  ← 分一の位
│  [5][6][7][8][9]             │
│                              │
│  [ ] 同伴  +¥3,000          │
│                              │
│  [キャンセル]      [作成]    │
└──────────────────────────────┘
```

### 1-3. 動作仕様

- **客種選択**: 新規/R/正規の3ボタン。タップで即反映（ステップ送りなし）
  - R: サブボタン（チケ有/無）が下に表示。タップで確定
  - 新規: セット料金ボタンが下に表示。タップで確定
  - デフォルト: 正規（createInitialStateと同じ）
- **入店時間**: ButtonTimePickerをインライン埋め込み（3段階ボタン）
  - 時→分十の位→分一の位の順に表示切替
  - デフォルト: 20:00
- **同伴**: トグル1つ。デフォルトOFF
- **イベントフラグ等**: ダイアログには含めない。作成後に基本情報タブから設定
- **[作成]**: ダイアログを閉じ、伝票を作成、会計タブをアクティブに
- **[キャンセル]**: ダイアログを閉じ、伝票を作成しない
- **[✕]**: キャンセルと同じ

### 1-4. 発動タイミング
- 「+ 伝票追加」タップ時にダイアログ表示
- ウィザードモード（isWizardActive）は廃止

### 1-5. コピーとの関係
- 「コピーして追加」は従来通りSlipCopyModalを使用（ダイアログ不要）

## 2. LOページ 2列レスポンシブ対応

### 2-1. 概要
タブレット等の画面幅768px以上で、テーブルカードを**2列表示**する。  
左→右→左→右の順で配置。

### 2-2. 実装

```css
/* デフォルト（スマホ）: 1列 */
.lo-grid { display: flex; flex-direction: column; gap: 1rem; }

/* タブレット以上: 2列 */
@media (min-width: 768px) {
  .lo-grid { display: grid; grid-template-columns: 1fr 1fr; gap: 1rem; }
}
```

Tailwindクラス: `flex flex-col md:grid md:grid-cols-2 gap-4`

### 2-3. テーブル順序
- 通常通り配列順に配置（左→右→左→右…）
- A, B-1 | B-2, C-1 | C-2, D | E-1, E-2 | E-3
- 各カードは2列時に均等幅
