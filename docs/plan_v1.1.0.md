## 1. LOモード料金表示切替

### 現状
LOページの内訳セクションは常に **現在時刻の料金** を表示。
ヘッダーには「現在」と「閉店」の2つの金額が並んでいるが、内訳は現在料金のみ。

### 変更内容
- LOページ内訳の上部に **「現在」「ラストまで」の切替タブ** を追加
- 「現在」: `result.currentTotal` + `result.breakdown`（現行動作）
- 「ラストまで」: `closingResult.totalPrice` + `closingResult.breakdown`（閉店時の計算結果）
- **デフォルト切替**: 現在時刻が **23:45以降** → 「ラストまで」をデフォルト選択
- 23:45未満 → 「現在」をデフォルト選択

### UI設計
```
┌─────────────────────────────┐
│ [現在]  [ラストまで]         │  ← タブ切替（23:45以降はラストまでがアクティブ）
├─────────────────────────────┤
│ セット料金(延長込)  ¥8,000  │
│ カン x3            ¥2,100  │
│ ...                         │
│ 合計 (税込)       ¥18,300  │
└─────────────────────────────┘
```

## 2. アップデート通知画面

### 現状
vite-plugin-pwaの`autoUpdate`で自動更新されるが、ユーザーに変更内容が通知されない。

### 変更内容
- アプリ内にバージョン番号を保持（`src/version.ts`）
- 初回アクセス時にlocalStorageの保存バージョンと比較
- 不一致の場合 → **アップデート通知モーダル** を表示
- 表示後、localStorageに現在バージョンを保存

### UI設計
```
┌─────────────────────────────┐
│        ✦ アップデート         │
│         v1.1.0              │
├─────────────────────────────┤
│                             │
│ • LOモード料金切替を追加     │
│ • テーブル3×3グリッド表示    │
│ • 全卓全伝票一括削除         │
│ • リアルタイム時刻更新       │
│ • 会計時刻のタップ入力対応   │
│ • 同伴ボタンをメニュー外へ   │
│ • PWA即時更新               │
│                             │
│      [ OK ]                 │
└─────────────────────────────┘
```

### バージョン管理ルール
- localStorageキー: `app-version`
- 比較: `localStorage値 !== version.ts値` → モーダル表示
- OKタップ → `localStorage.setItem('app-version', CURRENT_VERSION)`

## 3. リアルタイム時刻更新

### 現状
`App.tsx`: `setInterval(updateTime, 60000)` → **60秒間隔**。
最悪59秒のラグが発生し、延長切替タイミングを逃す可能性がある。
さらにLOページの各伝票は`currentTime`が更新されず料金が古いまま。

### 変更内容
- 更新間隔を **60秒 → 10秒** に短縮
- `UPDATE_ALL_CURRENT_TIME` アクションを追加し **全テーブル全伝票** の時刻を一括更新
- LOページの料金もリアルタイム反映

### 対象ファイル
- `src/App.tsx`: setInterval間隔変更 + `UPDATE_ALL_CURRENT_TIME` dispatch
- `src/hooks/useMultiTableCalculator.ts`: 新アクション追加

## 4. 会計時刻タップ入力

### 現状
現在時刻はテキスト表示のみ。会計の締め時刻を手動指定する方法がない。

### 変更内容
- 「現在時刻: HH:MM」をタップ → **time入力に切り替え**
- 入力確定 → その時刻での会計金額を表示（一時的なオーバーライド）
- ✕ボタン → リアルタイムに戻る
- オーバーライド中は自動更新を一時停止

### UI設計
```
通常表示:
  現在時刻: 23:15          ← タップで入力モードへ

入力モード:
  会計時刻: [23:30]  [✕]   ← time入力 + 戻るボタン
```

## 7. PWA即時更新（新規）

### 現状
vite-plugin-pwaの`registerType: 'autoUpdate'`で自動更新が有効だが、
PWAから開いた場合に古いSWキャッシュが使われ、新バージョンが反映されるまでに
2回のアクセスが必要になる場合がある。

### 変更内容
- vite-plugin-pwaの `registerSW` を使ってSW更新検知 → **即座にページリロード**
- `vite.config.ts` に `selfDestroying: false` を明示
- `src/App.tsx` で `registerSW({ onNeedRefresh })` を呼び、SW更新時に自動リロード

### 動作フロー
1. ユーザーがPWAを開く
2. ブラウザがバックグラウンドでsw.jsの更新をチェック
3. 新しいSWが検出される → `onNeedRefresh` コールバック発火
4. `updateSW(true)` を即座に呼び出し → SWアクティベート + ページリロード
5. リロード後のアプリは最新版 → アップデート通知モーダル表示

### 対象ファイル
- `src/App.tsx`: registerSW import + onNeedRefresh処理

## 8. LOテーブル3×3グリッド表示（新規）

### 現状
LOページのテーブルは **縦1列のカード** で順番に並ぶ。
9テーブル（A, B-1, B-2, C-1, C-2, D, E-1, E-2, E-3）あるため
スクロール量が多く全体把握が困難。

### 変更内容
- テーブル一覧を **3×3のグリッドレイアウト** で表示
- 各セルにテーブル名 + 伝票数 + 合計金額をサマリー表示
- セルタップで伝票詳細を展開表示（アコーディオン or モーダル）
- モバイル対応: 画面幅に応じて3列 or 2列

### UI設計
```
┌─────────┬─────────┬─────────┐
│ A       │ B-1     │ B-2     │
│ 2伝票   │ 1伝票   │ 0伝票   │
│ ¥45,200 │ ¥18,300 │   ---   │
├─────────┼─────────┼─────────┤
│ C-1     │ C-2     │ D       │
│ 1伝票   │ 3伝票   │ 0伝票   │
│ ¥22,100 │ ¥67,500 │   ---   │
├─────────┼─────────┼─────────┤
│ E-1     │ E-2     │ E-3     │
│ 0伝票   │ 1伝票   │ 0伝票   │
│   ---   │ ¥12,000 │   ---   │
└─────────┴─────────┴─────────┘

↓ テーブルタップで展開

┌─────────────────────────────┐
│ ▼ C-2（3伝票）              │
│  ① 新規 | 入店21:00        │
│     現在: ¥18,300           │
│  ② 正規 | 入店20:00 | 同伴 │
│     現在: ¥32,200           │
│  ③ R(チケ有) | 入店22:00   │
│     現在: ¥17,000           │
└─────────────────────────────┘
```

### 対象ファイル
- `src/components/LOPage.tsx`: グリッドレイアウト + セル→展開のUI

## 9. 全卓全伝票削除ボタン（新規）

### 現状
伝票削除は1伝票ずつ or 1テーブル内の全削除のみ。
営業終了後にリセットするには各テーブルを巡回して削除が必要。

### 変更内容
- LOページ上部に **「全卓リセット」ボタン** を追加
- タップ → 確認ダイアログ「全テーブルの全伝票を削除しますか？」
- 確定 → 全テーブルの全伝票を一括削除
- `CLEAR_ALL_SLIPS` アクションをuseMultiTableCalculatorに追加

### UI設計
```
◆ LO（ラストオーダー一覧）  [全卓リセット]
```

### 対象ファイル
- `src/hooks/useMultiTableCalculator.ts`: `CLEAR_ALL_SLIPS` アクション追加
- `src/components/LOPage.tsx`: ボタンUI追加
